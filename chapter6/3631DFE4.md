# **Problem 2(Intermediate):**   C**onnection Pool Memorylessness** 

##### **\[Condition]**

##### Kim Woo hyun is currently debugging a complex C++ backend connection pool designed for a newly deployed Soongsil University data repository. The continuous arrival of database queries to the primary backend server is perfectly modeled by a stochastic Poisson process with a verified arrival rate of $\\lambda = 2$ queries per millisecond. During an intensive hardware stress test, the system diagnostic monitor displays that the server has already waited continuously for 1 full millisecond without receiving a single query. Observing this latency, Woo-hyun becomes concerned that an influx of incoming queries is statistically "overdue" and will inevitably arrive in the immediate fraction of a second.

##### **\[Problem]**

##### Given the absolute certainty that the connection pool has already waited 1 millisecond with zero query arrivals, what is the exact probability that the server will wait at least 1 more millisecond before the very next database query arrives?

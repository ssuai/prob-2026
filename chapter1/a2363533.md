Topic: Bayes' Rule and the False Positive Paradox
CS Context: Machine Learning / Automated Bug Detection

You are developing a Static Analysis Tool to detect a specific critical 
"memory leak" bug in source code. 

* Historical data: This bug exists in only 1% of the files.
* Tool Performance:
  - Sensitivity: If a file has the bug, the tool detects it (Positive) 
    with probability 0.98.
  - False Positive Rate: If a file does not have the bug, the tool 
    mistakenly reports a bug (Positive) with probability 0.05.

TASK:
You run the tool on a randomly selected file, and it returns a POSITIVE 
result. Provide a step-by-step derivation to calculate the probability 
that the file actually contains the memory leak.

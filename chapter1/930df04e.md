# Problem 3 (Hard)

A company has three servers \(S_1, S_2,\) and \(S_3\) that handle incoming login requests.

- \(50\%\) of all login attempts go through \(S_1\)
- \(30\%\) go through \(S_2\)
- \(20\%\) go through \(S_3\)

The probability that a login attempt is malicious depends on the server:

- \(P(M \mid S_1) = 0.01\)
- \(P(M \mid S_2) = 0.03\)
- \(P(M \mid S_3) = 0.05\)

where \(M\) is the event that a login attempt is malicious.

The company uses an intrusion detection system. The system behaves as follows:

- if a login attempt is malicious, it raises an alert with probability \(0.95\)
- if a login attempt is not malicious, it raises an alert with probability \(0.04\)

Let \(A\) be the event that the system raises an alert.

Answer the following questions.

## (a)

Find \(P(M)\), the overall probability that a randomly selected login attempt is malicious.

## (b)

Find the overall probability that the system raises an alert.

## (c)

Given that the system raised an alert, find the probability that the login attempt came from server \(S_3\).

## (d)

Find the probability that a login attempt is actually malicious given that the system raised an alert.

## (e)

Compare the likelihood that the system raises an alert when a login attempt is malicious with the posterior probability that a login attempt is malicious when the system raised an alert. Explain clearly why these two probabilities are different, and state what this reveals about intrusion detection in rare-event settings.


# Hard (Partition)

**Problem Title**
* Partitioning of Distinct Servers into Redundant Clusters

**Problem Statement**
* A system administrator at Soongsil's server farm needs to distribute 4 distinct high-performance servers $\{S_1, S_2, S_3, S_4\}$ into exactly 2 identical, non-empty clusters for a high-availability setup.
* Calculate the number of unique ways to partition these 4 distinct items into 2 unlabelled subsets.

**Mandatory Solution Protocol**
* To receive full credit, **students must systematically illustrate** all 7 unique partition cases by categorizing them into two structural types:
    1. **3:1 Isolation**: Extracting a single server (singleton) from the group (e.g., $\{S_1, S_2, S_3\} \cup \{S_4\}$).
    2. **2:2 Equal Split**: Dividing the servers into two balanced pairs (e.g., $\{S_1, S_2\} \cup \{S_3, S_4\}$).
* Use simple grouping diagrams to visualize each configuration. This approach helps verify that all possible states are accounted for without relying on abstract formulas.

**Professor's Insight**
* The primary goal of this problem is to cultivate a systematic approach to state-space analysis.
* By manually categorizing possible partitions into specific structures—specifically the 3:1 isolation and 2:2 equal split—students learn to identify and enumerate all valid configurations without relying solely on abstract formulas.
* This visual method reinforces the fundamental principle of 'indistinguishability' in group partitioning; students must recognize that because the clusters are identical, the order of subsets does not matter.
* Such intuition is crucial for architecting fault-tolerant systems, where verifying server groupings case-by-case ensures that no critical redundancy configuration is overlooked.

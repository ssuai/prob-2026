**\*\*\[Hard Problem: C++ Movie Rating System Capacity]\*\***

**A C++ application manages a database of user ratings for movies.** 

**The system processes a daily batch of n = 10,000 independent** 

**user rating entries. The processing time for a single entry** 

**follows an unknown distribution, but profiling shows a mean** 

**time of μ = 2 ms and a standard deviation of σ = 0.5 ms.**



**Part 1 (Capacity Planning with CLT):**

**Using the Central Limit Theorem, calculate the minimum total** 

**allocated time T (in ms) the system needs for the daily batch** 

**to ensure that the probability of the total processing time** 

**exceeding T is at most 2.5%.** 

**(Hint: You may use the standard normal CDF value Φ(1.96) ≈ 0.975)**



**Part 2 (Estimation Confidence with WLLN/Chebyshev):**

**The system administrator wants to estimate the true mean** 

**processing time within a strict margin of error of ε = 0.05 ms.** 

**Using Chebyshev's inequality (as applied in the Weak Law of** 

**Large Numbers), prove whether the current sample size of** 

**10,000 entries is sufficient to guarantee that the estimate** 

**is accurate with a probability of at least 0.99.**

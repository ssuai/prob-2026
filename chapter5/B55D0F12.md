**[Intermediate Problem: Algorithm Collision Analysis]**
A hash table algorithm resolves collisions. The time X (in ms) 
it takes to resolve a collision has a mean of μ = 10 ms and a 
variance of σ² = 4 ms². A request is considered "timed out" 
if resolving the collision takes 20 ms or longer.

1. Calculate the upper bound for the probability of a timeout 
   using Markov's Inequality.
2. Calculate the upper bound for the probability of a timeout 
   using Chebyshev's Inequality.
3. If the backend processes a batch of 5,000 independent 
   collisions, what is the maximum expected number of timed-out 
   requests, assuming you rely on the tighter (more accurate) 
   mathematical bound?

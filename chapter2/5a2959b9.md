# **Problem 2 (Intermediate): Expectation, Mean, and Variance**

## **Theme: Variance of Algorithm Execution Time in Load Balancing**

* An asynchronous data processing algorithm deployed on a high-availability AI cluster routes incoming tensor data through one of two analytical branches. Depending entirely on the CPU cache state and branch predictor, the algorithm requires either exactly 2 milliseconds or exactly 4 milliseconds to complete execution. Real-time system logs generated by an online profiling tool show that the algorithm is equally likely to take either execution path (Probability = 0.5 each).
* To ensure strict Quality of Service (QoS) guarantees and avoid heavy-tailed execution delays, the data center engineers need to mathematically evaluate the jitter (dispersion) of the processing times. Let $Y$ be the execution time of the algorithm in milliseconds. Calculate the mathematical Variance of $Y$.



##### **Extra Credit Features**

* **Professor's Insight:**
* In advanced computer science and distributed architecture, merely minimizing average algorithmic execution time ($E$) is wholly insufficient. Real-world systems require highly predictable behavior. High execution variance leads directly to network jitter, thread synchronization failures, and degraded user experience. Using the moments expression $E - (E)^2$ dramatically simplifies the computational complexity of evaluating algorithmic stability without requiring complex integral calculus.

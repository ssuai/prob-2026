# Problem 2: Chess Engine Tactical Analysis

**Problem Statement:**
You are testing a Java-based chess engine simulation. Let $X$ represent the number of candidate moves the algorithm generates at a specific depth, where $X \in \{1, 2, 3\}$. Let $Y$ represent the number of those candidate moves that lead to an immediate tactical advantage, where $Y \in \{0, 1, 2\}$.

The joint probability mass function $P_{X,Y}(x, y)$ for the algorithm's performance is given in the table below:

| | $Y=0$ | $Y=1$ | $Y=2$ |
| :--- | :---: | :---: | :---: |
| **$X=1$** | 0.10 | 0.05 | 0.00 |
| **$X=2$** | 0.15 | 0.20 | 0.05 |
| **$X=3$** | 0.10 | 0.20 | 0.15 |

**Questions:**
1. Find the marginal probability that the engine generates exactly 2 candidate moves, $P(X=2)$.
2. Given that the engine found at least one move leading to a tactical advantage ($Y \ge 1$), what is the probability that exactly 3 candidate moves were generated ($X=3$)?

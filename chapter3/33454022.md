# **Problem3 (Hard): Expected Value Modeling in Serverless Cloud Billing**

#### **Topic Category: Expectation and Variance of Continuous RVs**

#### **Difficulty Level: Hard**

#### **Domain Context: Serverless Computing / Cloud Billing**

##### **Problem Context and Statement**

* A highly scaled enterprise API backend is hosted entirely on a serverless compute platform. The billing architecture for this platform abandons traditional server uptime metrics; instead, it generates financial charges based on a continuous function of the exact execution time required to process every individual network request.
* Due to highly variable network payload sizes, cache miss ratios, and asynchronous database retrieval latencies, the exact execution time for this specific microservice fluctuates unpredictably. Exhaustive telemetry indicates that the execution time, denoted by the continuous random variable $X$ (measured in standard seconds), strictly follows a Continuous Uniform Distribution bounded securely between a minimum floor of 1 second and a maximum hard timeout limit of 5 seconds.
* The probability density function (PDF) representing the execution time $X$ is defined as:
* $$f\_X(x) = \\begin{cases} \\frac{1}{4}, \& \\text{if } 1 \\le x \\le 5 \\\\ 0, \& \\text{otherwise} \\end{cases}$$
* The cloud provider's billing engine calculates the final financial cost of a single function invocation based on a base network routing fee added to a dynamic per-second compute multiplier. The total cost, $C$, measured in micro-dollars, is given by the continuous linear transformation:
* $$C = g(X) = 2X + 1$$

##### **Questions for the Student:**

##### **Part A:**

* Calculate the Expected Value (the exact average anticipated cost) of a single serverless function invocation, $E\[C]$. You must exclusively use the continuous Expected Value Rule for functions of a random variable, demonstrating the full integral derivation to arrive at the solution.

##### Part B:

* During a standard operational window, the platform processes exactly 10,000 completely independent client requests. Assuming the mathematical properties of expectation hold, what is the total expected cost for running this microservice during this window?
